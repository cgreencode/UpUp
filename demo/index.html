a tiny script that makes sure your site is always there for your users

(even when they're on a plane over the Atlantic, or 20,000 leagues under the sea)

Your users are no longer sitting at their desktops, with a wire connecting them to the internet. They are visiting your site from a phone in an elevator, from their tablet on the subway, or on their laptop while they sit cramped into seat 25 E.

Luckily, modern browsers support technologies that let you serve your content to your users, even when they are offline.

UpUp makes using all of this modern magic as simple as calling one command.


      <script src="upup.min.js"></script>
      <script>
        UpUp.start({
          'content-url': 'offline.html' // show this page to offline users
        });
      </script>

UpUp is powerful enough to support much more advanced use cases, even allowing an offline experience that includes fully functional single page applications, with data customized to the current user.


      <script src="upup.min.js"></script>
      <script>
        UpUp.start({
          'content-url': 'schedule.html?user=TalAter', // show this to the current user when he is offline
          'assets': [                // define additional assets needed to display the offline content:
            'img/logo.png',          // such as images,
            'css/offline.css',       // custom stylesheets,
            'schedule.TalAter.json', // personalized data for the current user,
            'js/angular.min.js',     // javascript libraries,
            'contacts.xlsx'          // and more.
          ]
        });
      </script>

It's time to start the offline-first revolution.
